# Model Routing and Cost Control

> [!abstract] Definition
> The policy for deciding which model handles which class of work based on quality needs, latency, privacy, and cost.

## Typical Split

- stronger models for planning, architecture, or high-risk review
- cheaper models for routine checks, classification, or summarization
- local models where privacy or cost dominates

# Memory and Retrieval Strategy

> [!abstract] Definition
> The rules for what context agents keep, how they retrieve it, and how they avoid being overloaded by irrelevant repo state.

## Good Strategy Components

- stable canonical notes
- repository maps
- recent task memory
- retrieval by symbol, ownership, or contract boundary
- explicit discard rules for stale context

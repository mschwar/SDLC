# Contract-Driven Development

> [!abstract] Definition
> Define interfaces, data shapes, behavioral expectations, and boundaries before implementation so agents can work inside explicit constraints.

## Good Contract Types

- API schemas
- data validation rules
- state transitions
- acceptance criteria
- infrastructure expectations

## When To Add It

Use contracts when ambiguity produces repeated defects, parallel teams need stable edges, or agent context is too weak for implicit conventions.

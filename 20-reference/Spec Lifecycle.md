# Spec Lifecycle

> [!abstract] Definition
> The set of transitions a spec goes through from draft intent to implementation input to historical record.

## Typical Stages

- proposed
- clarified
- approved
- implemented
- revised
- superseded

## Why It Matters

Without a lifecycle, specs become dead documents or hidden mutable prompts.

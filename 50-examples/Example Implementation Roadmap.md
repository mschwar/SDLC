# Example Implementation Roadmap

> [!summary] Use
> This is a practical order for adopting the vault in a new repository.

## Phase 1 - Baseline Repo Guardrails

1. Copy [[40-templates/AGENTS Template]]
2. Add local quality checks from [[40-templates/Quality Checks Template]]
3. Add [[40-templates/CI Workflow Template]]
4. Enable branch protection

## Phase 2 - Structure and Review

1. Add architecture and decision notes
2. enforce conventional commits
3. adopt draft PR visibility
4. require evidence in PRs

## Phase 3 - Scale and Autonomy

1. add contracts where ambiguity hurts
2. improve retrieval with repository maps
3. introduce observability-backed feedback loops
4. add higher-autonomy patterns only after the core path is reliable

# Architecture and Process Change Playbook

> [!summary] Goal
> Change the workflow, architecture, or governance model without causing invisible drift between the documentation system and the actual operating process.

## Steps

1. Define the change and why the current state is insufficient.
2. Identify which canonical notes and templates are affected.
3. Update the decision record first.
4. Update prompts or policies alongside the docs, not later.
5. Stage the rollout in the smallest safe slice.
6. Validate the new process on one task or one repo area.
7. Archive superseded notes immediately.
8. Update playbooks and definition-of-done requirements if behavior changed.

## Exit Criteria

- documentation, prompts, and actual behavior agree
- superseded notes are archived
- the approval path is still clear

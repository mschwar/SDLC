# Release Playbook

> [!summary] Goal
> Move validated work into a runtime environment with explicit versioning, rollout control, observability, and rollback readiness.

## Steps

1. Confirm the release criteria and definition of done.
2. Confirm the versioning or release-train decision.
3. Choose the deployment strategy.
4. Verify observability and alerting coverage.
5. Verify rollback readiness.
6. Apply required human approval gates.
7. Execute the rollout.
8. Observe the release window and capture follow-up actions.
